03 Vision LLM — image to text
GPT-4o Vision or Claude Sonnet

Vision LLM — factual description

Each ad image is forwarded once. The LLM describes only what is literally visible — no interpretation, no hooks, no judgment. Prompt explicitly forbids assumptions. For unclear details: "not visible" instead of guessing.

"Woman around 30 years old on a beach lounger. Silver bracelet on left wrist. Two children playing in the sand in the background. Sunny day, warm tones. No text visible in image."

06 MeTTa / PLN — reasoning
Hyperon — probabilistic logic

MeTTa / PLN — reasons continuously over all atoms

Detects patterns · computes confidence scores · surfaces opportunities · learns from each new campaign cycle. Brand atom is outside the analysis — only becomes active at output.

Example patterns PLN detects

"lifestyle + weekend + winter" → ROAS 3.8 · conf: 0.79 · n=14

"urgency + holiday -3 days" → CTR high but conv low · conf: 0.71

"Valentine day -10 start" → 40% higher ROAS than day -3 · conf: 0.84

"silver bracelet + beach + summer" → best seasonal combination · conf: 0.88

08 PLN — fetch ad specifications
PLN query on Atomspace

PLN decides what gets generated

User asks: "10 ads for silver bracelet." PLN searches the Atomspace for all atoms relevant to this product, season and day. Sorts by confidence score. Selects the strongest proven combinations plus a set of underexplored variations with potential.

What PLN returns — 10 ad specs

Ads 1–4 · proven combinations · conf >0.80 · lifestyle + model + weekend

Ads 5–7 · strong variations · conf 0.65–0.79 · lifestyle + product shot

Ads 8–10 · underexplored potential · conf 0.50–0.64 · test new combination

Per ad spec

Contents of one spec

  • visual_style · setting
  • hook_type · emotional_angle
  • product focus · context
  • day/season context
  • brand colors · tone
  • image_ref (best match)
  • confidence score
  • base atoms (traceable)
10 ad specs → LLM for text generation
09 LLM — prompt generation (text only)
Text generation · no creative decisions

LLM converts ad specs into image prompts

The LLM receives a tightly structured PLN spec for each ad. Its only task: translate that spec into a detailed image prompt. All creative choices have already been made by PLN. The LLM only fills in the language.

Example: spec → prompt

Spec · lifestyle · beach · model · silver bracelet · warm gold · summer

Prompt · "Woman relaxing on sunlit beach, close-up of silver bracelet on wrist, warm golden tones, natural light, editorial style, soft background blur, luxury feel"

Also generated

Ad copy per ad

  • headline based on spec
  • body copy · tone of voice
  • CTA from brand atom
  • everything traceable to spec
10 prompts → verification loop
10 Verification loop 1 — prompt check
Automated check

Check: does the prompt follow the spec?

  • every spec field present?
  • no added elements?
  • brand colors correct?
  • tone of voice correct?
  • no hallucinations in copy?
PLN variation check

Check: are variations correctly steered?

  • every prompt unique?
  • variations within PLN bounds?
  • confidence distribution correct?
  • no overlap between ads?
On failure

Reject → regenerate

  • failing prompt back to LLM
  • with the exact mismatch reason
  • max 2 retries per prompt
  • on persistent failure: flag for user
✓

Approved — all 10 prompts meet the spec. No hallucinations. Variations within PLN bounds. Ready for image generator.

approved prompts → image generator
11 Image generator
Generation · e.g. DALL·E 3 or Midjourney API

Image generator receives approved prompts

Each approved prompt + image_ref (style reference from Atomspace) is sent to the image generator. The image_ref drives the style — the prompt drives the content. Output: 10 images, one per ad spec.

Input per generation

What the generator receives

  • detailed image prompt
  • image_ref as style reference
  • format · ratio (Meta specs)
  • brand color palette as guidance
generated images → verification loop 2
